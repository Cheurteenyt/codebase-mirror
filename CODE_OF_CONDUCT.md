# Code of Conduct

Codebase Memory is an open technical project. Participation must remain
respectful, constructive, and focused on improving the software and its
documentation.

## Expected behavior

- discuss ideas and evidence without attacking people;
- welcome questions, different experience levels, and good-faith corrections;
- keep reviews specific, actionable, and proportionate to the risk;
- protect private information, credentials, and unpublished vulnerability
  details;
- stop an interaction when a maintainer asks for de-escalation.

Harassment, discrimination, threats, doxxing, sexualized conduct, repeated
personal attacks, and deliberate disruption are not accepted in repository
issues, pull requests, discussions, or related project spaces.

## Enforcement

Maintainers may edit or remove content, close or lock an interaction, reject a
contribution, or temporarily or permanently restrict participation when this
code is violated. Decisions should consider context, severity, repetition, and
the safety of affected participants.

For a non-sensitive concern, open a concise repository issue. For a matter
that should not be public, request a private contact channel through the
maintainer's GitHub profile without posting the incident details. Security
vulnerabilities must instead follow [SECURITY.md](SECURITY.md).

Reports will be reviewed as promptly and impartially as the project's current
maintainer capacity allows. Retaliation against a good-faith reporter is itself
a violation of this code.

# Repository audit — 2026-07-15

## Scope

This audit covers the V2 backend, CLI, MCP server, embedded Graph UI,
packaging, Windows operation, and GitHub workflow surface. Reproducible product
defects require a repository fix and a regression test, as formalized in
`AGENTS.md`.

The audit was performed on Windows from the exact requested base,
`9453537d837960e6d3d8c62075fe36a95b34e423`. Linux CI remains authoritative
for the POSIX durability and shell fixtures.

## Outcome

- No P0 blocker was found.
- Source installation, parallel indexing, MCP, packaged UI, and the local Graph
  UI work on Windows.
- The confirmed UI security, project-routing, process-lifecycle, Windows
  portability, MCP precision, and update-automation defects are fixed and
  covered by regression tests.
- The Graph UI is now lazy-loaded, responsive, bounded to 1,000 rendered nodes,
  backed by a substantially cheaper layout query, and protected by a bounded
  idle project-store registry.
- Two P1 limitations remain: atomic publication is merged but inactive, and the
  complete backend suite is not platform-aware on Windows.

Measured performance and transfer-byte results are recorded in
[`PERFORMANCE_TOKEN_UI_AUDIT_2026-07-15.md`](PERFORMANCE_TOKEN_UI_AUDIT_2026-07-15.md).

## Principal changes

- Added repository-level bug-fix and Windows portability rules in `AGENTS.md`.
- Corrected Windows worker startup and added strict worker-count validation.
- Replaced the graph-status shell pipeline with a cross-platform Git call.
- Confined project SQLite paths and rejected unsafe cross-platform filenames.
- Aligned the Node.js, CI, and Docker runtime contract; documented npm 10/11
  compatibility while keeping npm 10.9.0 only as an authoring hint.
- Hardened loopback HTTP, WebSocket, child-process ownership, shutdown, and
  multi-project store identity. Idle UI stores use a four-project LRU budget;
  each code reader and human store retains a 64 MiB and 8 MiB SQLite page-cache
  budget respectively.
- Added an explicit `ui --allowed-root <paths...>` allowlist for Control-tab
  browsing/indexing outside the home directory or the selected project's
  indexed root.
- Made SQLite project deletion a prepare/rollback operation and preserved exact
  Windows file identity with bigint device/inode comparisons.
- Normalized vault traversal output to portable forward-slash
  `obsidian_path` values so nested Obsidian imports work on Windows.
- Let Windows `taskkill /T` use the remaining global UI shutdown deadline after
  escalation instead of applying a separate 250 ms cutoff.
- Made full discovery the safe default and retained fast discovery as an
  explicit performance mode. Full discovery includes hidden application and
  test trees; the V1-compatible fast policy does not blanket-exclude `test` or
  `tests` directories.
- Added compact, bounded MCP outputs and a JSON whitespace transport-byte
  benchmark. MCP initialization is ordered strictly, active human-memory counts
  are explicit, and bounded coverage scans label partial/lower-bound results.
- Reworked `/api/layout` around bounded bulk SQL reads and a balanced graph
  sample; the UI pauses simulation off-screen, lazy-loads heavy routes, and
  classifies test provenance before structural shape so Hide tests is complete.
- Enabled bounded grouped Dependabot updates for Actions, V2, Graph UI, and
  Docker, with package and container smoke jobs plus focused Windows UI
  lifecycle/security coverage in CI.

passing checks and the unresolved Windows full-suite failures:

  npm artifact, 1,146,338 compressed bytes, with required runtime files present
  and source/tests/private configuration absent.

  no horizontal overflow or browser console warnings/errors.
- Dependency audit: zero known vulnerabilities in both npm projects.
- Dependency update check: no compatible-range update remains; ten direct
  major migrations are isolated for separate review.

The local Docker client is available but its Linux daemon is not, so Docker
runtime validation is delegated to the required Linux CI job. The inactive
generation-publication benchmark remains intentionally red on Windows because
its POSIX mode and directory-`fsync` contract has not been redesigned for ACLs;
the same smoke benchmark passes under Ubuntu/WSL.

## Recommended execution order

1. Merge only after all required Linux, Windows, package, and Docker checks pass
   and the protected-branch approval is present.
2. Activate the generation publisher through the production indexer and reader
   cutover, then run the integrated R169 crash/concurrency gates.
3. Classify the full test matrix into portable and POSIX-only suites; port
   product behavior tests and gate true POSIX fixtures explicitly.
4. Upgrade major dependencies in isolated PRs, including the Node floor change
   required by jsdom 29.
5. Add an exact-tag release workflow with checksums and provenance.
